# Verifying Releases

Every release artifact — binaries and Docker images — can be independently verified before use. This page covers the three verification mechanisms available.

## Release Channels

| Channel | Tag | Trigger | Verification |
|---|---|---|---|
| **Mainnet** | semver (e.g. `0.3.0`) | Git tag push | SHA256SUMS |
| **Testnet latest** | `testnet-latest` | Push to `testnet` branch | SHA256SUMS |
| **Testnet nightly** | `testnet-nightly` | Nightly cron after E2E passes | SHA256SUMS |
| **Docker (release)** | `:latest`, `:<tag>` | GitHub Release created | SLSA provenance attestation |
| **Docker (dev)** | `:dev`, `:dev-<sha>` | Push to `testnet` branch | SLSA provenance attestation |
| **Docker (nightly)** | `:testnet-nightly` | Nightly cron after E2E passes | SLSA provenance attestation |

## Verifying Binary Checksums

Every GitHub Release includes a `SHA256SUMS` file containing the checksums of all release binaries. The [install script](../install.sh) performs this verification automatically.

To verify manually after downloading a binary:

```console
# Download the binary and checksums
TAG="testnet-latest"  # or a semver tag like 0.3.0
curl -fLO "https://github.com/inference-labs-inc/subnet-2/releases/download/${TAG}/sn2-miner-linux-x86_64"
curl -fLO "https://github.com/inference-labs-inc/subnet-2/releases/download/${TAG}/SHA256SUMS"

# Verify (Linux)
sha256sum --check --ignore-missing SHA256SUMS

# Verify (macOS)
shasum -a 256 --check --ignore-missing SHA256SUMS
```

Both should print `sn2-miner-linux-x86_64: OK`. A mismatch indicates the binary was tampered with or corrupted in transit.

## Verifying Docker Image Attestations

Docker images published to `ghcr.io` include [SLSA build provenance](https://slsa.dev/) attestations generated by GitHub Actions during the build. These attestations are signed by Sigstore and pushed to the container registry alongside the image.

Verification requires the [GitHub CLI](https://cli.github.com/) (`gh >= 2.49.0`):

```console
# Verify a specific image tag
gh attestation verify oci://ghcr.io/inference-labs-inc/subnet-2:latest \
  --owner inference-labs-inc

# Verify the dev image
gh attestation verify oci://ghcr.io/inference-labs-inc/subnet-2:dev \
  --owner inference-labs-inc

# Verify the nightly image
gh attestation verify oci://ghcr.io/inference-labs-inc/subnet-2:testnet-nightly \
  --owner inference-labs-inc
```

Successful output includes the Sigstore bundle, the workflow that produced the image, and the source commit SHA.

## What the Auto-Updater Verifies

When `sn2-miner` or `sn2-validator` auto-updates (see [versioning](versioning.md)), the update mechanism:

1. Downloads the platform-specific binary from the latest GitHub Release
2. Downloads the `SHA256SUMS` file from the same release
3. Computes the SHA-256 hash of the downloaded binary and compares it to the expected value
4. Only replaces the running binary if the checksum matches

Auto-update can be disabled with `--no-auto-update`.
